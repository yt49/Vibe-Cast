# 要件定義書：vibe-cast-tsurudai

**プロジェクト名：** vibe-cast-tsurudai（AIに仕事奪われてみたチャンネル - 収録・編集・配信ツール）  
**作成日：** 2026-04-02  
**対象ユーザー：** 2名（ホスト：Windows / ゲスト：macOS）

---

## 1. プロダクト概要

二人のリモートポッドキャスト収録から、AI編集・記事化・配信までを一気通貫で行うWebアプリ。  
ZencastrやRiverside.fmのような高音質リモート収録に、AI編集と配信機能を統合した、二人専用のプライベートツール。

---

## 2. ユーザー・環境

| 項目 | 内容 |
|------|------|
| ユーザー数 | 2名（固定） |
| ホスト環境 | Windows + マイク |
| ゲスト環境 | macOS + マイク |
| アプリ形態 | Webアプリ（ブラウザで動作、インストール不要） |
| ストレージ | Google Cloud Storage（二人共有） |
| 想定収録テーマ | AIと仕事・働き方（カジュアルなトーン） |

---

## 3. 機能要件

### 3-1. リモート収録機能

- ブラウザ間でWebRTCを使ったリアルタイム音声通話
- 外部サービス（Zoom・Discord等）に依存しない自走型
- 各ユーザーの音声を**個別トラックとして録音**（ミックスダウンせず分離保存）
- 収録音声はGoogle Cloud Storageに自動アップロード

### 3-2. リアルタイム効果音ボタン

- 収録中にワンクリックで効果音を鳴らせるボタンUI
- 効果音はあらかじめ登録済みのファイルから選択
- 効果音のタイムスタンプを記録し、編集時に自動挿入できるよう保持

### 3-3. AI自動編集

| 機能 | 内容 |
|------|------|
| フィラー音除去 | 「えー」「あー」「えっと」などを自動検出・カット |
| BGM自動挿入 | オープニング・エンディングにBGMを自動付与 |
| 効果音挿入 | 収録中に記録したタイムスタンプ位置に効果音を合成 |
| 音声書き出し | 編集済みの音声ファイル（MP3/WAV）を生成 |

### 3-4. 文字起こし・記事化

| 機能 | 内容 |
|------|------|
| 文字起こし | 話者ごとに分離したトランスクリプト生成 |
| 要約生成 | AIによるエピソードサマリー（200〜400字程度） |
| note記事生成 | 要約をベースにnote投稿用の記事を自動生成 |
| note自動投稿 | ホストのnoteアカウントに下書きまたは公開投稿 |

**記事のトーン：** カジュアル・親しみやすい、AIと仕事をテーマにした読み物スタイル

### 3-5. 配信フロー

- 編集・記事化が完了すると「配信しますか？」確認画面を表示
- 承認後、ポッドキャスト配信プラットフォームへ投稿（プラットフォームは後日決定）
- RSSフィード生成機能を実装（Spotify・Apple Podcasts等への対応基盤）

---

## 4. 非機能要件

| 項目 | 内容 |
|------|------|
| クロスプラットフォーム | Windows・macOS両方でChrome/Edgeから動作 |
| 音質 | 各自のトラックをロスレスまたは高ビットレートで収録 |
| プライバシー | 録音データは二人の共有GCSのみに保存、第三者アクセスなし |
| 認証 | Googleアカウント認証（GCS連携と兼用） |
| コスト | 個人利用の範囲でGoogle Cloud無料枠・低コスト運用を優先 |

## 5. 画面構成（概要）

```
[ホーム]
  └─ 新規収録開始 / 過去エピソード一覧

[収録画面]
  ├─ 通話接続・音声レベル表示
  ├─ 録音開始・停止ボタン
  └─ 効果音パネル（ボタン群）

[編集画面]
  ├─ 波形表示・タイムライン
  ├─ AI編集実行ボタン（フィラー除去・BGM挿入）
  └─ プレビュー再生

[記事化画面]
  ├─ 文字起こし表示・編集
  ├─ 要約・記事プレビュー
  └─ note投稿ボタン

[配信画面]
  └─ 「配信しますか？」確認 → 配信実行
```

## 7. データの流れ

```
【収録フェーズ】
  ブラウザA（Windows）               ブラウザB（macOS）
       │                                    │
  自分の音声をMediaRecorder録音       相手の音声をMediaRecorder録音
       │                                    │
       └──────── WebRTC通話（P2P）─────────┘
       │                                    │
       ↓                                    ↓
  host_track.webm                    guest_track.webm
       │                                    │
       └──────────────┬─────────────────────┘
                      ↓
            Google Cloud Storage
            /episodes/{id}/raw/
              ├── host_track.webm
              └── guest_track.webm

【編集フェーズ】
            Google Cloud Storage
                      │
                      ↓
            backend/audio_processor.py（ffmpeg）
              ├── フィラー音除去（えー・あー）
              ├── BGM合成（オープニング・エンディング）
              └── 効果音挿入（収録時タイムスタンプ位置）
                      │
                      ↓
            /episodes/{id}/edited/
              └── episode_final.mp3

【記事化フェーズ】
            episode_final.mp3
                      │
                      ↓
            backend/transcriber.py（Whisper）
              └── トランスクリプト（話者分離）
                      │
                      ↓
            backend/article_generator.py（Gemini API）
              ├── エピソード要約（200〜400字）
              └── note投稿用記事（本文・タグ）

【配信フェーズ】
            記事・音声ファイル
                      │
              「配信しますか？」確認UI
                      │
                      ↓
            backend/note_publisher.py → note.com投稿
            backend/publish.py       → RSSフィード更新
```

## 8. 技術スタック（案）

| レイヤー | 技術候補 |
|----------|----------|
| フロントエンド | React + TypeScript |
| リアルタイム通話 | WebRTC（PeerJS or mediasoup） |
| バックエンド | Python（FastAPI）または Node.js |
| 音声処理 | ffmpeg + Whisper（文字起こし） |
| AI要約・記事生成 | Gemini API（音声・長文コンテキスト処理に強み） |
| ストレージ | Google Cloud Storage |
| 認証 | Google OAuth 2.0（GCS・Gemini API と認証統一） |
| note投稿 | note.com API（または非公式スクレイピング） |
| ホスティング | ローカル起動 または Cloud Run（最小構成） |
