## 2. Cloud Run CORS エラー修正

### 問題

```
Access to fetch at 'https://vibe-cast-backend-...run.app/ai/transcribe'
from origin 'https://vibe-cast-tsurudai.web.app' has been blocked by CORS policy
```

### 原因

```python

# 修正後
allow_origins=[
    "https://vibe-cast-tsurudai.web.app",
    "https://vibe-cast-tsurudai.firebaseapp.com",
    "http://localhost:5173",
    "http://localhost:3000",
],
allow_credentials=True,
```

## 4. GCS 署名付きURL（Signed URL）アップロード方式の実装

### 背景

65MB 超の音声ファイルを `POST /audio/upload`（バックエンド経由）でアップロードすると、リクエストボディがバックエンドを通過するためタイムアウトや不安定が発生。GCS の署名付きURL方式に切り替えることで、ブラウザ → GCS の直接転送を実現。

### 変更内容

**新規作成**

- `backend/app/services/gcs_client.py` — `generate_upload_url(bucket_name, blob_name)` を実装

**追加エンドポイント**

- `GET /audio/upload-url?episode_id=xxx&role=host`
  - PUT メソッド・15分有効の署名付きURL を返す
  - 保存パス: `episodes/{episode_id}/{role}.webm`

### Cloud Run での署名付きURL生成の注意点

Compute Engine の default credentials は `sign_bytes` を持たないため、`blob.generate_signed_url()` を直接呼ぶとエラーになる。`google.auth.iam.Signer` → `google.oauth2.service_account.Credentials` を経由して署名する必要がある。

```python
signer = Signer(request, credentials, credentials.service_account_email)
signing_credentials = SACredentials(signer=signer, ...)
blob.generate_signed_url(version="v4", ..., credentials=signing_credentials)
```

**必要な IAM 権限:** `roles/iam.serviceAccountTokenCreator`（`gcloud projects add-iam-policy-binding` で付与済み）

## 6. Whisper 25MB 制限対応：ffmpeg による自動チャンク分割

### 問題

65MB の WebM ファイルを Whisper API に投げると 413 エラー（上限 25MB）。

### 解決策（第1版：pydub）

`pydub` + `AudioSegment.from_file()` で分割 → **メモリ不足で 503 に**

65MB の圧縮 WebM は pydub でデコードすると生 PCM として **700MB〜1GB** に膨張するため、4GB RAM でも足りなかった。

### 解決策（第2版：ffmpeg 直接呼び出し）

```python
subprocess.run([
    "ffmpeg", "-y",
    "-ss", str(start_sec),   # ファイル全体を展開せずシーク
    "-t",  str(chunk_sec),
    "-i",  src_path,
    "-ac", "1",              # モノラル
    "-ar", "16000",          # 16kHz（Whisper最適）
    "-b:a", "32k",
    chunk_path,
])
```

- `AudioSegment.from_file()` を使わないのでメモリ消費 ≒ 0
- 各チャンクを 32kbps / 16kHz / モノラル MP3 で出力 → 1時間分で約14MB（25MB 上限を大幅に下回る）
- `Dockerfile` に `apt-get install -y ffmpeg` を追加
- `requirements.txt` から `pydub` を削除

### 分割ロジック

```
n_chunks = ceil(ファイルサイズ / 20MB)
chunk_sec = 総尺 / n_chunks

→ 65MB, 67分の場合: n_chunks=4, chunk_sec=16.75分
→ 各 MP3 チャンク: 約 4MB（25MB 上限に対して余裕十分）
```

### useGCS をブラウザ直接転送に切り替え

```typescript
// 旧: ブラウザ → バックエンド(100MB) → GCS（タイムアウトリスク）
fetch(BACKEND_URL/audio/upload, { body: formData })

// 新: 署名付きURLを取得してブラウザから GCS へ直接 PUT
const { upload_url } = await fetch(BACKEND_URL/audio/upload-url).json();
await fetch(upload_url, { method: 'PUT', headers: { 'Content-Type': 'audio/webm' }, body: blob });
```

### Host/Guest 分離の保証確認

`transcribe_episode` の構造を `try/except/finally` で各ロール完全独立化。

| ケース | 動作 |
|--------|------|
| host のみ存在 | guest をスキップ、host を処理 |
| guest のみ存在 | host をスキップ、guest を処理 |
| 一方が失敗 | `except` でキャッチ → `None`、もう一方は継続 |
| 両方 None | `ai.py` 側で 404（transcriber はエラーを上げない） |

## 8. article_generator: google-genai 移行・JSON堅牢化・長文対応

### 背景

`POST /ai/summarize` で `json.decoder.JSONDecodeError` が発生。Gemini が JSON をマークダウンコードブロックで囲んで返すケースがあり、`json.loads()` が失敗していた。あわせて `google.generativeai`（非推奨）から公式最新パッケージへ移行。

### 変更内容

| 項目 | 変更前 | 変更後 |
|---|---|---|
| ライブラリ | `google-generativeai` / `genai.GenerativeModel` | `google-genai` / `genai.Client` |
| JSON抽出 | `json.loads(text)` のみ | `_extract_json()` でコードブロック除去後に解析 |
| JSONエラー | 500 エラー伝播 | `JSONDecodeError` キャッチ → フォールバック dict を返す |
| 長文対応 | 無制限 | 話者ごと 8,000 字でトランケート（合計最大 16,000 字） |
| 空入力 | `or "（音声なし）"` | `_truncate()` で None・空白・長文を統一処理 |

### JSON 抽出ロジック

```python
def _extract_json(text: str) -> str:
    match = re.search(r"```(?:json)?\s*([\s\S]+?)\s*```", text)
    if match:
        return match.group(1)
    return text.strip()
```

Gemini が ` ```json ... ``` ` で囲んだ場合でも内部 JSON のみ取り出せるようにした。

## 今日のトラブルと教訓

### `allow_origins=["*"]` + `allow_credentials=True` は無効

CORS 仕様上、`credentials` モードのリクエストに対してワイルドカードは使えない。明示的なオリジンリストが必須。

### Secret Manager のローカル認証は ADC で十分

`service-account.json` をローカルに置く必要はない。`gcloud auth application-default login` 一発で解決。

### pydub は大きな圧縮音声には使えない

65MB の webm/opus を `AudioSegment.from_file()` でロードすると生 PCM として RAM に 700MB〜1GB 展開される。音声の分割は必ず ffmpeg のサブプロセス呼び出しで行う。

### `gcloud run deploy --source .` はディレクトリに注意

ホームディレクトリから実行すると `node_modules` 内の Windows symlink でクラッシュする。常に `backend/` ディレクトリ内から実行する。

### Cloud Run での 404 デバッグは openapi.json が早い

`curl .../openapi.json` でルート一覧を確認すれば、「ルーティング問題か」「ロジックの 404 か」を即座に切り分けられる。
