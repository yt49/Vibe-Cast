## 2. 履歴タブ（HistoryPage）の新設

- トランスクリプト・サマリー（Journey含む）の表示
- 削除ボタン（確認ダイアログあり）

### 新規作成ファイル

**`frontend/src/pages/HistoryPage.tsx`**

各エピソードの状態を `Record<episode_id, EpisodeState>` という辞書で管理：

```typescript
type EpisodeState = {
  expanded: boolean;
  transcript: Transcript | null;
  transcribing: boolean;
  summary: Summary | null;
  summarizing: boolean;
  deleting: boolean;
  editingName: string;
  savingName: boolean;
};
```

### stale closure バグと修正

「カードをタップすると一瞬開いてすぐ閉じる」バグがあった。

**原因：** `patchState` の中で古いReact state（`states`）を参照していた。

```typescript
// ❌ バグあり（getState(id)が古いstatesを参照）
function patchState(id, patch) {
  setStates((prev) => ({ ...prev, [id]: { ...getState(id), ...patch } }));
}
```

**修正：** `setStates` コールバック内で `prev` を使う。

```typescript
// ✅ 修正後
function patchState(id, patch) {
  setStates((prev) => ({
    ...prev,
    [id]: { ...(prev[id] ?? defaultState), ...patch },
  }));
}
```

展開トグルも同様の問題があったため、functional update 内に閉じ込めた：

```typescript
const handleExpand = async (id) => {
  let wasExpanded = false;
  setStates((prev) => {
    const cur = prev[id] ?? defaultState;
    wasExpanded = cur.expanded;
    return { ...prev, [id]: { ...cur, expanded: !cur.expanded } };
  });
  if (wasExpanded) return;
  // 以降: transcript/summaryをフェッチ
};
```

## 3. 文字起こし・要約結果をGCSに永続保存

### 問題

文字起こし・要約はReactのstateにしか保存されていなかったため、
ページをリロードしたり履歴タブから見たりすると消えていた。

### 解決策

バックエンドで処理完了後にGCSのJSONとして保存する。

| ファイル | 内容 |
|---------|------|
| `episodes/{id}/transcript.json` | `{ host: "...", guest: "..." }` |
| `episodes/{id}/summary.json` | Gemini の要約オブジェクト |
| `episodes/{id}/meta.json` | `{ name: "..." }` |

**`backend/app/routers/ai.py` の変更：**

```python
def _save_to_gcs(blob_name, data):
    blob = storage.Client().bucket(BUCKET_NAME).blob(blob_name)
    blob.upload_from_string(json.dumps(data, ensure_ascii=False), content_type="application/json")

# transcribeの最後
_save_to_gcs(f"episodes/{episode_id}/transcript.json", payload)

# summarizeの最後（episode_idが渡された場合のみ）
if req.episode_id:
    _save_to_gcs(f"episodes/{episode_id}/summary.json", summary)
```

RecordingPageから要約を呼ぶとき `episode_id` を渡し忘れていたバグも修正：

```typescript
// ❌ 修正前（GCSに保存されない）
body: JSON.stringify({ host: transcript.host, guest: transcript.guest })

// ✅ 修正後
body: JSON.stringify({ episode_id: episodeId, host: transcript.host, guest: transcript.guest })
```

## 4. バックエンド: episodes ルーターの新設

**`backend/app/routers/episodes.py`** を新規作成。

| エンドポイント | 内容 |
|-------------|------|
| `GET /episodes` | エピソード一覧（id + name）を返す |
| `GET /episodes/{id}/audio?role=host\|guest` | 音声をストリーミング配信 |
| `GET /episodes/{id}/transcript` | 保存済みトランスクリプトを返す |
| `GET /episodes/{id}/summary` | 保存済みサマリーを返す |
| `POST /episodes/{id}/meta` | エピソード名を保存 |
| `DELETE /episodes/{id}` | エピソード以下の全ファイルを削除 |

### 音声配信：署名付きURLからストリーミングへ

Cloud Run では追加IAM設定なしに署名付きURLが生成できず500エラーになった。

**解決策：** バックエンドがGCSから読んでブラウザに直接ストリーミング配信する。

```python
def iterfile():
    with blob.open("rb") as f:
        while chunk := f.read(1024 * 256):
            yield chunk

return StreamingResponse(iterfile(), media_type="audio/webm")
```

フロントは `<audio src={バックエンドURL} controls />` とするだけでよい。

## 5. 録音IDのタイムゾーン修正

`toISOString()` がUTC時刻を返すため日本時間より9時間ずれていた。

```typescript
// ❌ 修正前（UTC）
return new Date().toISOString().slice(0, 19).replace(/[-T:]/g, '');

// ✅ 修正後（JST = UTC+9）
const now = new Date(Date.now() + 9 * 60 * 60 * 1000);
return now.toISOString().slice(0, 19).replace(/[-T:]/g, '');
```

## 6. エピソード名をつける機能

### 録音タブ（RecordingPage）

アップロード完了後に名前入力フィールドが表示される。
Enterまたは「保存」で `POST /episodes/{id}/meta` に送信、GCSに保存。

### 履歴タブ（HistoryPage）

カードを展開すると一番上に名前入力フィールドが表示される。
保存するとカードのタイトルも即座に更新される。

```typescript
const handleSaveName = async (id) => {
  await fetch(`${BACKEND_URL}/episodes/${id}/meta`, {
    method: 'POST',
    body: JSON.stringify({ name: s.editingName.trim() }),
  });
  // 成功したらepisodesリストのnameも更新
  setEpisodes((prev) => prev.map((e) => e.id === id ? { ...e, name: ... } : e));
};
```

名前があればタイトルに表示、なければ録音日時にフォールバック：

```tsx
<div>{name || formatEpisodeId(id)}</div>
```

## 今日のトラブルと教訓

### トラブル1：Windowsでgcloudコマンドの改行ができない

Macでは `\` で改行できるが、Windowsのコマンドプロンプトでは使えない。

```
# ❌ Windows では動かない
gcloud run deploy vibe-cast-backend \
  --source .

# ✅ 1行で書く
gcloud run deploy vibe-cast-backend --source . --region asia-northeast1 --project vibe-cast-tsurudai
```

### トラブル2：ビルドと手動デプロイの順番

`firebase deploy` だけ実行してもビルドが古いままの場合がある。
**必ず `npm run build` → `firebase deploy` の順で実行する。**
JSファイル名のハッシュ（例: `index-QoA7uHHf.js`）が変わっていれば新しいコードが反映されている。

### トラブル3：バックエンドとフロントエンドの更新タイミングのずれ

フロントが新しい形式 `{id, name}` を期待しているのにバックエンドが古い形式 `["id1"]`（文字列配列）を返すと、destructuringで `id = undefined` になり `undefined.length` でクラッシュ。

対策として、フロント側で両形式を吸収するようにした：

```typescript
const normalized = raw.map((e) =>
  typeof e === 'string' ? { id: e, name: '' } : e
);
```

### トラブル4：要約がGCSに保存されない

RecordingPageから `/ai/summarize` を呼ぶ際に `episode_id` を渡し忘れていた。
バックエンドは `episode_id` がない場合は保存をスキップする設計だったため、
履歴タブで要約が表示されなかった。
