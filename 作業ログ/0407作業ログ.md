## 1. タブ切替時の録音継続バグ修正（#16 / PR #24）

### 問題

`App.tsx` が条件レンダリングでタブを切り替えていたため、録音タブを離れると `RecordingPage` がアンマウントされ、`useRecorder`・`useWebRTC` フックが破棄されて録音・WebRTC 接続が切れていた。

```tsx
// ❌ 修正前（タブ切替でアンマウントされる）
{tab === 'record' ? <RecordingPage /> : <HistoryPage />}
```

### 解決策

`display: none` による表示切替に変更し、全タブを常時マウントする方式に修正。

```tsx
// ✅ 修正後（常時マウント、CSSで隠す）
<div style={{ display: tab === 'record' ? 'block' : 'none' }}><RecordingPage /></div>
<div style={{ display: tab === 'history' ? 'block' : 'none' }}><HistoryPage /></div>
```

これにより、別タブを見ながら録音を継続できる。今後タブを追加する際も同じパターンを使う。

## 3. arXiv パイプライン PoC（#18 / PR #25）

### 作成ファイル

`backend/app/trend_pipeline/poc_arxiv_pipeline.py`

### 処理フロー

```
arXiv API → 直近1週間 cs.AI 論文 30件取得
    ↓
Gemini 2.5 Flash → アブストラクトからキーワード最大8件抽出
    ↓
正規化（小文字・複数形統一）+ 共起カウント
    ↓
trend_output.json（nodes / edges 形式）出力
```

### 出力 JSON 形式（React Flow で直接読み込める）

```json
{
  "nodes": [{"id": "LLMs", "weight": 4}, ...],
  "edges": [{"source": "LLMs", "target": "Reinforcement Learning", "weight": 2}, ...]
}
```

### PoC 結果

30論文から LLMs・AI agents・Reinforcement Learning・Hallucination の4ノード、エッジ2本を生成。ノード数が少ないのは論文数 30件・閾値 MIN_NODE_WEIGHT=2 のため。論文数を増やせば密なネットワークになる。

### ハマりポイント

- `backend/.env` が存在せず、GCP Secret Manager から GEMINI_API_KEY を取得して `.env` を生成した（Cloud Run は Secret Manager から直接読むため .env 不要だった）
- `gemini-2.0-flash` が廃止されていた → `gemini-2.5-flash` に変更（既存バックエンドと統一）
- `google.generativeai` パッケージが非推奨 → `google-genai`（`from google import genai`）に移行
